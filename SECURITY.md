# Security Policy

## Supported Versions

Only the latest published release receives security fixes.

## Reporting a Vulnerability

Do not open a public GitHub Issue for security vulnerabilities.

Report them privately by emailing the maintainer or using GitHub's private vulnerability reporting feature. Include:

- A description of the vulnerability
- Steps to reproduce it
- Potential impact
- Any suggested fix, if available

You will receive an acknowledgement within 72 hours. If the issue is confirmed, a fix will be prioritised and a patched release published. You will be credited in the release notes unless you prefer otherwise.

## Scope

This policy covers the extension source code and its published `.vsix` packages. It does not cover Apache Tomcat itself, the Java runtime, or any third-party tooling the extension invokes.
